Introduction back to ToC

This vocabulary aims to represent the metadata associated with a FAIR testing resource, along with its provenance (i.e., how the result was run by a test), the metrics associated with that test (i.e., how it is supposed to be measured) and the quantitative assessment (i.e., FAIR score) resulting from its interpretation in a given context. The rest of the document introduces the main definitions, namespaces and examples of the version 1.3.0 of the specification. In order to ease the adoption of the FTR, the following documents complement this specification: - A [JSON-LD Conext](https://github.com/OSTrails/FAIR_testing_resource_vocabulary/tree/main/release/1.3.0/ftr_context.jsonld) which translates all namespaces to simple keys (see examples below) - An [OpenAPI specification](https://github.com/OSTrails/FAIR_testing_resource_vocabulary/tree/main/release/1.3.0/api) with a basic description of the API for a FAIR assessment tool - [SHACL and ShEx shapes](https://github.com/OSTrails/FAIR_testing_resource_vocabulary/tree/main/release/1.3.0/SHACL) to ease validation of metrics, tests, test results, etc.
