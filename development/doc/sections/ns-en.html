Other relevant FTR specifications back to ToC

In order to ease the adoption of the FTR, the following documents complement this specification: - A [JSON-LD Conext](https://github.com/OSTrails/FAIR_testing_resource_vocabulary/tree/main/release/1.3.0/ftr_context.jsonld) which translates all namespaces to simple keys (see examples below) - An [OpenAPI specification](https://github.com/OSTrails/FAIR_testing_resource_vocabulary/tree/main/release/1.3.0/api) with a basic description of the API for a FAIR assessment tool - [SHACL and ShEx shapes](https://github.com/OSTrails/FAIR_testing_resource_vocabulary/tree/main/release/1.3.0/SHACL) to ease validation of metrics, tests, test results, etc.
